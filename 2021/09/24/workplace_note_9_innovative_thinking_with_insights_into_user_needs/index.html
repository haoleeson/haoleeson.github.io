职场笔记：洞察用户需求的创新思维

编写 2021-09-24 浏览：

创新是为了解决某个问题，达成某个目标
创意 != 创新

  • Idea：一个想法
  • 创意：一个新奇的想法
  • 创新：一个新奇的、有价值的、可行的想法

创新：

  • 技术可行
  • 商业可行
  • 用户需求

课程目的：

  • 洞察问题/用户需求本质
  • 提供创新思维工具
  • 介绍快速验证原则

1. 搞清楚要解决的问题究竟是什么？

1.1. 同理心洞察

洞察问题/需求本质

  • 一问：
    • 通过访谈，了解到用户的真实想法/需求，
    • 发现问题的本质
    • Need：用户真正需要的是什么？
  • 二看：
    • 把用户的行为作为他生活的一部分，在真是的场景下进行观察
    • User：目标用户有哪些行为和表现？（目标用户是谁？）
  • 三体：
    • 体验到用户所体验到的所有，挖掘客户潜在的真正感受、想法、思考
    • Insight：用户有这样的需求的内心情感是？

目标：如何明确 Need 让 User 满足 Insight。

3. 创新着陆

四种解决方案原型

  • 理论原型
  • 虚拟原型（伪造解决方案）
  • 最小化可行原型（MVP）
  • 最小化卓越产品

三种测试

  • 惊喜测试
  • NPS 测试
  • 付款测试

创新着陆步骤

  • 解决方案（头脑风暴） == 功能和解决方案变化 ==>>
  • 理论原型 == 惊喜测试 ==>>
  • 虚拟原型 == 惊喜、净推荐值和付款测试 ==>>
  • 最小化可行原型 == 付款或净推荐值测试 ==>>
  • 最小化卓越产品
